# Malloc Lab: 동적 저장 공간 할당기 작성

> 원문: CS 213, Fall 2001 — *Malloc Lab: Writing a Dynamic Storage Allocator*  
> 과제 배정·마감 등 일정은 기관별로 다를 수 있습니다.

---

## 1. 소개

이 실습에서는 C 프로그램용 **동적 저장 공간 할당기**, 즉 여러분만의 `malloc`, `free`, `realloc` 루틴을 구현합니다. 설계 공간을 창의적으로 탐색하고, **정확하고 효율적이며 빠른** 할당기를 만드는 것을 권장합니다.

## 2. 진행 방식

## 4. 실습에서 구현할 내용

동적 할당기는 `mm.h`에 선언되고 `mm.c`에 정의되는 다음 **네 함수**로 구성됩니다.

```c
int mm_init(void);
void *mm_malloc(size_t size);
void mm_free(void *ptr);
void *mm_realloc(void *ptr, size_t size);
```

제공된 `mm.c`는 **가능한 한 단순하지만 동작은 맞는** malloc 패키지입니다. 이를 출발점으로 위 함수들(필요하면 **비공개 `static` 함수** 추가)을 수정하여 아래 의미를 따르게 하세요.

- **`mm_init`**: 애플리케이션(구현을 평가할 trace 기반 드라이버)이 `mm_malloc`, `mm_realloc`, `mm_free`를 부르기 **전에** `mm_init`을 호출합니다. 초기 힙 영역 할당 등 필요한 초기화를 수행합니다. 초기화에 문제가 있으면 **-1**, 아니면 **0**을 반환합니다.

- **`mm_malloc`**: 최소 `size`바이트의 **페이로드**를 가리키는 포인터를 반환합니다. 할당된 블록 전체는 힙 안에 있어야 하고, **다른 할당 블록과 겹치면 안 됩니다**. 표준 C 라이브러리(libc)의 `malloc`과 비교하며, libc는 페이로드 포인터를 **8바이트 경계**에 맞추므로 여러분 구현도 **항상 8바이트 정렬**을 지켜야 합니다.

- **`mm_free`**: `ptr`이 가리키는 블록을 해제합니다. 반환값은 없습니다. `ptr`는 이전에 `mm_malloc` 또는 `mm_realloc`이 돌려준 포인터이고 **아직 `mm_free`되지 않은 것**일 때만 동작이 보장됩니다.

- **`mm_realloc`**: 최소 `size`바이트의 할당 영역을 가리키는 포인터를 반환하며, 다음을 만족합니다.  
  - `ptr == NULL`이면 `mm_malloc(size)`와 동일합니다.  
  - `size == 0`이면 `mm_free(ptr)`와 동일합니다.  
  - `ptr != NULL`이면 이전에 `mm_malloc` 또는 `mm_realloc`의 결과여야 합니다.  
  `ptr`가 가리키던 **옛 블록**의 크기를 `size`로 바꾼 **새 블록**의 주소를 반환합니다. 새 주소는 구현·내부 단편화·요청 크기에 따라 옛 주소와 같을 수도, 다를 수도 있습니다.  
  **새 블록 내용**은 옛 블록과 **둘 중 작은 크기만큼** 동일합니다. 나머지는 미초기화입니다. (예: 옛 블록 8바이트·새 블록 12바이트면 앞 8바이트는 동일, 뒤 4바이트는 미초기화.)

이 의미는 libc의 `malloc`, `realloc`, `free`와 맞춥니다. 자세한 설명은 셸에서 `man malloc`을 참고하세요.

## 5. 힙 일관성 검사기

동적 할당기는 **타입 없는 포인터 조작**이 많아 올바르게 짜기 어렵습니다. 힙을 훑어 **일관성을 검사하는 힙 체커**를 두면 매우 도움이 됩니다.

검사 예시:

- 자유 리스트의 모든 블록이 “자유”로 표시되어 있는가?  
- 합병되지 않은 **인접 자유 블록**이 있는가?  
- 모든 자유 블록이 자유 리스트에 있는가?  
- 자유 리스트 포인터가 유효한 자유 블록을 가리키는가?  
- 할당 블록끼리 겹치는 것이 있는가?  
- 힙 블록 안의 포인터가 유효한 힙 주소를 가리키는가?

힙 체커는 `mm.c`의 `int mm_check(void)`로 구현합니다. 스스로 정한 불변 조건·일관성을 검사하고, 힙이 **일관되면 0이 아닌 값**, 일관되지 않으면 **0**을 반환합니다(원문: *nonzero if and only if your heap is consistent*). 목록에 없는 검사를 추가해도 되고, 모두 할 필요는 없습니다. `mm_check` 실패 시 **오류 메시지 출력**을 권장합니다.

이 검사기는 **개발 중 디버깅용**입니다. `mm.c`를 제출할 때는 **`mm_check` 호출을 제거**하세요. 호출이 남아 있으면 처리량이 떨어집니다. `mm_check`에는 **스타일 점수**가 반영됩니다. 무엇을 검사하는지 **주석으로 문서화**하세요.

## 6. 지원 루틴 (`memlib.c`)

`memlib.c`는 동적 할당기용 **메모리 시스템을 시뮬레이션**합니다. 다음 함수를 사용할 수 있습니다.

- **`void *mem_sbrk(int incr)`**: 힙을 `incr`바이트만큼 확장합니다. `incr`는 **양의 정수**이며, 새로 잡힌 힙 영역의 **첫 바이트**를 가리키는 범용 포인터를 반환합니다. Unix `sbrk`와 의미는 같지만, **`incr`는 양수만** 받습니다.

- **`void *mem_heap_lo(void)`**: 힙의 **첫 바이트** 주소.  
- **`void *mem_heap_hi(void)`**: 힙의 **마지막 바이트** 주소.  
- **`size_t mem_heapsize(void)`**: 현재 힙 크기(바이트).  
- **`size_t mem_pagesize(void)`**: 시스템 페이지 크기(바이트, Linux에서 흔히 4KB).

## 7. Trace 기반 드라이버

`malloclab-handout.tar`에 포함된 `mdriver.c`는 `mm.c` 패키지의 **정확성·공간 이용·처리량**을 검사합니다. **trace 파일** 집합으로 제어되며, 각 trace는 할당·재할당·해제 지시의 나열입니다. 드라이버와 trace는 **채점 시에도 동일하게** 사용됩니다.

`mdriver.c` 명령줄 인자:

- **`-t <tracedir>`**: 기본 trace 디렉터리 대신 `tracedir`에서 찾습니다.  
- **`-f <tracefile>`**: 기본 trace 집합 대신 **하나의 trace**만 사용합니다.  
- **`-h`**: 명령줄 인자 요약을 출력합니다.  
- **`-l`**: 학생 malloc 외에 **libc malloc**도 실행·측정합니다.  
- **`-v`**: 각 trace에 대해 **간단한 표 형태**로 성능을 출력합니다.  
- **`-V`**: 더 자세한 진단 정보를 출력합니다. 어떤 trace에서 실패하는지 찾을 때 유용합니다.

## 8. 프로그래밍 규칙

- `mm.c`의 **인터페이스는 바꾸지 마세요**.  
- **메모리 관련 라이브러리/시스템 호출**을 사용하지 마세요. `malloc`, `calloc`, `free`, `realloc`, `sbrk`, `brk` 및 이와 유사한 호출은 **코드에서 제외**합니다.  
- `mm.c`에 **전역 또는 `static` 복합 자료 구조**(배열, 구조체, 트리, 리스트 등)를 정의할 수 **없습니다**. 다만 **스칼라** 전역 변수(정수, 실수, 포인터 등)는 **허용**됩니다.  
- libc와 맞추기 위해 할당기는 **항상 8바이트 정렬** 포인터를 반환해야 하며, 드라이버가 이를 검사합니다.

## 9. 평가

규칙을 어기거나 버그로 드라이버가 죽으면 **0점**입니다. 그 외에는 대략 다음과 같이 채점됩니다.

- **정확성(20점)**: 드라이버 정확성 테스트를 통과하면 만점에 가깝게 부여하고, trace별로 **부분 점수**가 있을 수 있습니다.

- **성능(35점)**: 두 가지 지표를 씁니다.  
  - **공간 이용률**: 드라이버가 사용하는 메모리( `mm_malloc`/`mm_realloc`으로 할당되었고 아직 `mm_free` 안 된 양)와 할당기가 쓰는 **힙 크기**의 **피크 비율**입니다. 이상적으론 **1**에 가깝고, 단편화를 줄이는 정책이 필요합니다.  
  - **처리량(throughput)**: 초당 완료한 연산 수의 평균입니다.

드라이버는 **성능 지수 P**를 계산합니다. 공간 이용률과 처리량의 **가중합**입니다.

    P = wU + (1 - w) * min(1, T / T_libc)

여기서 U는 공간 이용률, T는 여러분의 처리량, T_libc는 같은 환경·기본 trace에서 libc malloc의 추정 처리량입니다. 기본적으로 **공간 이용에 더 큰 비중**을 두며 \(w = 0.6\)입니다. \(T_{\mathrm{libc}}\)는 드라이버에 상수(예: 600 Kops/s)로 넣을 수 있습니다.

메모리와 CPU 모두 비용이 크므로, **한쪽만 극단적으로** 최적화하기보다 **이용률과 처리량의 균형**을 맞추는 것이 좋습니다.

- **스타일(10점)**  
  - 함수로 잘 나누고 **전역 변수는 최소화**합니다.  
  - 파일 맨 앞에 **자유·할당 블록 구조, 자유 리스트 구성, 조작 방법**을 설명하는 헤더 주석을 둡니다.  
  - 각 함수 앞에 **무엇을 하는지** 설명하는 주석을 둡니다.  
  - `mm_check`는 **충실하고 문서화**되어야 합니다.  
  힙 체커와 코드 구조·주석에 각각 점수가 붙을 수 있습니다.
