# 테스트 방법
옵션 | 의미
--|--
-v |trace별 결과 요약
-V |더 자세한 로그
-a |mm.c 팀 정보 검사 생략
-l |libc malloc도 같이 돌림
-h |사용법 출력

# trace 디렉터리만 바꾸려면 
- (-f와 동시에 쓰면 -f가 우선이라 -t는 무시됨):
``` bash
./mdriver -t ./traces/
```

# gdb 사용법
## gdb 실행
```
gdb -q ./mdriver
```
## 프로그램에 인자 주고 실행
```
(gdb) run -V -f short1-bal.rep
```

## 바로 한번에 실행 + 종료 (스크립트처럼)
```
gdb -q ./mdriver -ex "run -V -f short1-bal.rep" -ex "quit"
```

## 크래시 났을 때 "어디서 죽었는지" 바로 보기
- 스택 트레이스
```
(gdb) bt
```
- 현재 프레임/상위 프레임 이동
```
(gdb) frame 0
(gdb) up
(gdb) down
```
- 현재 위치 소스 보기
```
(gdb) list
(gdb) list 120,170
```
## 브레이크포인트(멈추기)
- 함수에 걸기
```
(gdb) break mm_init
(gdb) break extend_heap
(gdb) break coalesce
```
- 파일에 걸기
```
(gdb) break mm.c:150
```
- 브레이크 목록/삭제/끄기
```
(gdb) info break
(gdb) delete 1
(gdb) disable 1
(gdb) enable 1
```
## 한 줄씩 실행
- 다음 줄로(함수 안으로 안 들어감)
```
(gdb) next
```
- 함수 안으로 들어가기
```
(gdb) step
```
- 현재 함수 끝까지
```
(gdb) finish
```
- 다음 브레이크까지 계속
```
(gdb) continue
```
## 값 보기/추적
- 변수/식 출력
```
(gdb) print bp
(gdb) print size
```
- 포인터를 주소로 보기(16진수)
```
(gdb) p/x bp
```
- 메모리 덤프(포인터 주변 확인)
    - (ex. bp 주변 16개 워드)
```
(gdb) x/16wx bp
```
- 자동 출력(멈출 때마다 보여줌)
```
(gdb) display bp
(gdb) undisplay
```
- segfault 원인 잡을 때 추천 루틴
```
- run ... 해서 죽이면 -> bt로 죽은 함수/라인 확인
- 그 함수에 break 걸고 다시 run ...
- 멈추면 next/step 로 진행하면서
    - 의심 포인터는 p/x ptr
    - 메모리는 x/16wx ptr
    - 이상한 값(예: 0x0, 0xdeadbeef, 너무 작은 값, 말이 안되는 정려)을 찾기
```
- 종료
```
(gdb) quit
```
